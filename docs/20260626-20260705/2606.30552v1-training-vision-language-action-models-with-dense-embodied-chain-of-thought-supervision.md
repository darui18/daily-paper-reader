## 摘要
跨实体迁移在视觉-语言-动作（VLA）模型中仍然具有挑战性，因为不同机器人平台的低级状态和动作空间存在根本性差异。我们观察到，操作背后的高级认知过程，包括场景感知、物体识别、任务规划和子任务分解，在很大程度上是跨实体共享的。基于这一观察，我们提出了ZR-0，一个拥有26亿参数的端到端VLA模型，该模型使用密集的具身思维链（ECoT）监督来对齐视觉-语言模型（VLM）中的跨实体表示。ZR-0采用双流架构：预训练的VLM（系统2）在训练期间生成结构化的ECoT推理，而基于扩散变换器的动作专家（系统1）通过流匹配生成连续的动作块。这两个组件通过交叉注意力耦合，并带有一个注意力掩码，该掩码限制动作专家仅使用输入提示特征，从而在推理时完全跳过ECoT生成，且没有任何性能损失。ZR-0在ProcCorpus-60M上进行预训练，这是一个大规模数据集，包含来自超过40万条轨迹的大约6000万帧（约1000小时），并带有覆盖96.8%帧的密集ECoT注释。我们在三个模拟基准上评估了ZR-0，涵盖单臂（LIBERO）、双臂（RoboTwin 2.0）和类人（RoboCasa GR-1 Tabletop）实体，以及在xArm平台上进行的真实世界实验，展示了在所有设置下的强大性能。代码和模型检查点可在https://github.com/RUCKBReasoning/ZR-0获取。

## Abstract
Cross-embodiment transfer in vision-language-action (VLA) models remains challenging because low-level state and action spaces differ fundamentally across robot platforms. We observe that the high-level cognitive process underlying manipulation, including scene perception, object identification, task planning, and sub-task decomposition, is largely shared across embodiments. Based on this observation, we present ZR-0, a 2.6 billion parameter end-to-end VLA model that uses dense Embodied Chain-of-Thought (ECoT) supervision to align cross-embodiment representations within the vision-language model (VLM). ZR-0 adopts a dual-stream architecture: a pre-trained VLM (System 2) generates structured ECoT reasoning during training, while a Diffusion Transformer-based action expert (System 1) produces continuous action chunks via flow matching. The two components are coupled through cross-attention, with an attention mask that restricts the action expert to input prompt features only, enabling ECoT generation to be entirely skipped at inference without any performance loss. ZR-0 is pre-trained on ProcCorpus-60M, a large-scale dataset comprising approximately 60 million frames (approximately 1,000 hours) from over 400K trajectories, with dense ECoT annotations covering 96.8% of all frames. We evaluate ZR-0 on three simulation benchmarks spanning single-arm (LIBERO), bimanual (RoboTwin 2.0), and humanoid (RoboCasa GR-1 Tabletop) embodiments, as well as real-world experiments on the xArm platform, demonstrating strong performance across all settings. Code and model checkpoints are available at https://github.com/RUCKBReasoning/ZR-0.

## 1. 论文简介和核心思想

**背景**：视觉-语言-动作（VLA）模型旨在学习通用机器人策略，但不同机器人的低层状态和动作空间异构，导致跨实体迁移困难。现有方法多采用零填充或统一动作空间等格式级手段，未能解决语义对齐问题。

**核心思想**：本文认为，操作中的高层认知过程（场景感知、任务规划、子任务分解等）是跨实体共享的。基于此，提出ZR-0模型，通过在训练阶段引入密集的“具身思维链”（ECoT）监督信号，驱动视觉-语言模型（VLM）学习与具体机器人无关的、可迁移的共享表征。推理时，该思维链被完全跳过，不增加额外延迟。

---

## 2. 论文发现的问题

- **低层空间异构**：不同机器人平台的自由度、控制接口、底座类型等差异巨大，导致状态和动作维度物理含义不同，无法直接共享。
- **格式级方案局限**：现有方法如零填充和归一化仅能在数据格式层面兼容，无法让模型学到真正共享的、具有语义的可迁移表征，模型容易过拟合到特定实体的模式。
- **认知过程可共享**：一个关键洞察是，尽管底层控制不同，但机器人在执行任务时的高层推理（如“我要抓那个杯子”）是跨实体一致的，这是迁移学习的关键但未充分利用的桥梁。

---

## 3. 解决方案

ZR-0提出一个双流架构，灵感来自“系统1/系统2”认知框架：

- **系统2（VLM）**：基于预训练的Qwen3-VL-2B，接收任务指令和图像，生成结构化的ECoT推理序列。训练时，通过ECoT监督（场景描述、进度评估、未来计划、待做动作、目标物体边界框、离散动作）来学习对齐的表征。
- **系统1（动作专家）**：基于扩散变换器（DiT），将VLM输出的特征和机器人状态作为条件，通过流匹配（Flow Matching）直接生成连续的动作块。
- **耦合与高效推理**：两系统通过交叉注意力连接。关键设计是**注意力掩码**：在训练时将动作专家的注意力限制在VLM的输入提示（指令+图像）特征上，而**不关注生成的ECoT文本**。这使得推理时只需VLM一次前向传播即可获得所有所需特征，完全跳过ECoT的文本解码，实现零额外推理开销。
- **训练数据**：在ProcCorpus-60M（约6000万帧，96.8%帧有ECoT标注）上进行预训练，并混合通用视觉-语言数据以保持VLM的通用能力。
- **训练目标**：联合优化ECoT的下一词预测损失和动作的流匹配损失，前者只更新VLM，后者同时更新动作专家和VLM。

## 4. 效果

**实验设置**：在三个模拟基准（单臂LIBERO、双臂RoboTwin 2.0、人形RoboCasa GR-1 Tabletop）和真实xArm平台上评估，使用相同的预训练检查点进行微调。评价指标为成功率（模拟）和任务进度分数（真实）。

**关键定量结果**：
- **LIBERO**：平均成功率97.8%，在长时域套件LIBERO-10上达96.4%，超越π0.5等基线。
- **RoboCasa GR-1 Tabletop**：平均成功率69.3%，超越先前最佳方法JoyAI-RA（63.2%）6.1个百分点。在拾取-放置类任务上提升显著，但在需要关闭柜门等低频率交互任务上表现较弱。
- **RoboTwin 2.0**：Clean与Randomized设置下平均成功率分别为88.70%和87.98%，性能与当前最优模型相当，且对场景随机化具有更强的鲁棒性（Randomized vs Clean下降仅0.72%）。
- **真实xArm**：平均任务进度分数76.0，显著优于π0.5（67.8）。在推方块（OCR推理，提升27.9分）、收拾桌面（长时域）等任务上优势明显。

**消融实验**：
- 在LIBERO上去掉ECoT监督（仅保留动作损失）后，平均成功率从97.8%降至95.7%，在LIBERO-10上下降3.8个百分点，证明ECoT对提升VLM表征质量至关重要。

## 5. 下一步计划

- **扩大预训练数据规模**：当前数据量（约1000小时）远低于领先VLA模型（超10000小时）。扩大覆盖更多操作基元的多样化数据，有望提升对低频技能（如关门）的迁移能力。
- **利用人类第一人称视频**：ECoT的结构化推理（场景描述、任务规划）与主体无关，可应用于人类操作视频（如Ego4D），为VLM提供更丰富的视觉和语义表征，解决机器人数据匮乏问题。
- **高效ECoT标注策略**：为每一帧生成密集ECoT计算成本高昂。未来可探索关键帧选择策略，在匹配密集标注效果的同时显著降低计算开销，使ECoT方法更易规模化。
